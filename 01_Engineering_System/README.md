# Проект: Инженерная система расчёта и визуализации параметров трубопровода

Этот проект демонстрирует сквозную работу системного / fullstack аналитика: от требований и процессов до модели данных, API и архитектуры, на примере инженерной расчётной системы.

---

## 1. Контекст проекта

### 1.1. Бизнес‑контекст

Компания разрабатывает систему, которая позволяет инженерам моделировать поведение трубопроводов (нефтегаз, мультифазные потоки) и анализировать результаты расчётов. Источник исходных данных — проекты из существующего специализированного ПО (например, OLGA), а вычислительное ядро — отдельная библиотека/движок на C++.

Бизнес‑цель: ускорить и упростить работу инженеров за счёт:
- удобного импорта существующих проектов,
- настройки параметров через UI,
- автоматизированного запуска расчётов,
- наглядной визуализации результатов,
- экспорта данных для отчётности и дальнейшего анализа.

### 1.2. Роль аналитика

В рамках этого проекта аналитик отвечает за:

- формализацию требований (бизнес, функциональные, нефункциональные);
- моделирование бизнес‑процессов (BPMN);
- описание пользовательских сценариев (Use Cases);
- проектирование модели данных и БД (ER, SQL);
- проектирование REST API для интеграции клиентских приложений и расчётного ядра;
- участие в проектировании архитектуры системы;
- подготовку технической документации.

---

## 2. Обзор функциональности системы

Система предоставляет пользователю (инженеру) следующие основные возможности:

1. **Импорт проекта из OLGA**
   - Загрузка файлов проекта.
   - Валидация структуры и версии.
   - Разбор сущностей: узлы, трубы, профили, параметры.
   - Сохранение данных в БД.

2. **Настройка параметров расчёта**
   - Выбор сценариев.
   - Изменение входных параметров (дебиты, давление, температура и т.д.).
   - Сохранение шаблонов расчётов.

3. **Запуск расчёта**
   - Формирование задания расчёта.
   - Отправка задания в расчётный модуль (C++).
   - Отслеживание статуса (в очереди, выполняется, завершено, ошибка).

4. **Визуализация результатов**
   - Отображение профилей давления/температуры по трассе.
   - Просмотр значений по узлам/сечениям.
   - Графики, таблицы.

5. **Экспорт результатов**
   - Выгрузка в CSV/табличный формат.
   - Формирование наборов данных для отчётности.
