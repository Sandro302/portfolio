# Use Cases — Engineering Calculation System

Список ключевых пользовательских сценариев системы расчёта и визуализации параметров трубопровода.

---

## UC01 — Import OLGA Project

**Цель:** Инженер импортирует существующий проект из OLGA в систему, чтобы работать с топологией и выполнять расчёты в единой среде.

**Актёры:**
- Primary: Engineer
- Secondary: External System (OLGA, как источник файлов)

**Результат:** В БД создан проект со структурой узлов и труб, сформирован лог импорта, пользователь видит статус и возможные предупреждения.

---

## UC02 — Run Calculation

**Цель:** Инженер запускает расчёт для выбранного проекта с заданными параметрами или шаблоном.

**Актёры:**
- Primary: Engineer
- Secondary: Calculation Engine (C++), Calculation Service

**Результат:** Создано задание расчёта, произведён расчёт, результаты сохранены и доступны для просмотра.

---

## UC03 — View Results

**Цель:** Инженер просматривает результаты последнего (или выбранного) расчёта в графическом и табличном виде.

**Актёры:**
- Primary: Engineer

**Результат:** Пользователь видит профиль давления/температуры по трассе, значения в узлах/сегментах и может анализировать поведение системы.

---

## UC04 — Export Results

**Цель:** Инженер экспортирует результаты расчёта в CSV/табличный формат для дальнейшего анализа или отчётности.

**Актёры:**
- Primary: Engineer

**Результат:** Сформирован и выгружен файл с результатами в выбранном формате.

---

## UC05 — Edit Calculation Parameters

**Цель:** Инженер изменяет входные параметры расчёта (дебиты, давления, температуры, режимы) и повторно выполняет расчёт.

**Актёры:**
- Primary: Engineer

**Результат:** Обновлён набор параметров, выполнен новый расчёт, результаты сохранены отдельно от предыдущих.

---

## UC06 — Manage Templates

**Цель:** Инженер создаёт и редактирует шаблоны расчётов, чтобы быстро запускать типовые сценарии.

**Актёры:**
- Primary: Engineer

**Результат:** В системе хранятся один или несколько шаблонов, которые можно применять при запуске расчётов.
